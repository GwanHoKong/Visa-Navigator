# VisaPath Analytics

Full-stack web app helping international students navigate the U.S. work visa process from OPT to STEM OPT to H-1B.

Features include visa guides with checklists, interactive timelines, company recommendations, and an H-1B sponsorship analytics tool powered by the modeling pipeline from `H1B-Sponsorship-Analytics`.

Live demo: [https://visanavigator-black.vercel.app](https://visanavigator-black.vercel.app)  
Backend API: [https://visa-navigator.onrender.com](https://visa-navigator.onrender.com)

## Portfolio Connection

This app is the **productized deployment layer** for [H1B Sponsorship Analytics](https://github.com/GwanHoKong/H1B-Sponsorship-Analytics), the companion data science project that cleans USCIS employer data, engineers leakage-controlled features, and validates the dual-track H-1B sponsorship models.

Together, the two projects show the full path from data science to product deployment:

1. **H1B Sponsorship Analytics:** research, data pipeline, EDA, feature engineering, and model validation.
2. **VisaPath Analytics:** full-stack product experience that exposes the model through visa planning workflows, company recommendations, and sponsorship risk insights.

## H-1B Analytics Model

The `/api/predict` endpoint uses a dual-track design from the H1B Sponsorship Analytics project:

- **High-risk screening:** Binary Logistic Regression estimates the probability that the selected employer/industry profile maps to a High Approval Risk Tier.
- **Expected approval rate:** XGBoost Regression estimates the expected employer-year approval rate.

The model artifacts are built from `h1b_engineered_features.csv`, which is generated by `H1B-Sponsorship-Analytics`.

### Personalization Status

The prediction form includes personal profile fields such as major, GPA, work experience, and internship experience. These fields are part of the planned personalization layer and are currently shown in the product experience for user context only.

The current model output is based on USCIS employer history: industry, state, employer filing patterns, sponsorship risk tier, and expected approval-rate signals. Personal profile features do not affect the model score yet.

Validation results for the artifacts currently committed here:

| Track | Metric | Value |
|---|---|---:|
| High-risk screening | ROC-AUC | 0.6211 |
| High-risk screening | Recall | 0.5168 |
| High-risk screening | Precision | 0.1048 |
| Approval-rate regression | RMSE | 0.0559 |
| Approval-rate regression | MAE | 0.0169 |
| Approval-rate regression | R squared | 0.8830 |

## Local Development

### Prerequisites

- Node.js 18+ and npm
- Python 3.11+ and pip

### Backend

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

The API will be available at `http://localhost:8000`. API docs are at `http://localhost:8000/docs`.

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The app will be available at `http://localhost:3000`. It reads the API URL from `NEXT_PUBLIC_API_URL`, defaulting to `http://localhost:8000/api`.

### Rebuild ML Artifacts

Copy the H1B Analytics engineered dataset into:

```text
data/h1b_engineered_features.csv
```

Then run:

```bash
cd ml
python build_analytics_artifacts.py
```

This regenerates:

- `backend/models/h1b_high_risk_model.joblib`
- `backend/models/h1b_approval_rate_model.joblib`
- `backend/models/model_meta.joblib`
- `backend/lookups/*.csv`

The local input CSV is ignored by git; only compact deployment artifacts are committed.

## Deployment

### Frontend on Vercel

1. Import this repo in Vercel.
2. Set the root directory to `frontend`.
3. Add `NEXT_PUBLIC_API_URL` with your backend API base URL, for example `https://your-backend.onrender.com/api`.
4. Deploy.

### Backend on Render

1. Create a Render Web Service from this repo.
2. Set the root directory to `backend`.
3. Use `pip install -r requirements.txt` as the build command.
4. Use `uvicorn main:app --host 0.0.0.0 --port $PORT` as the start command.
5. Set `ALLOWED_ORIGINS` to your Vercel domain and local dev origin, for example `https://visa-navigator.vercel.app,http://localhost:3000`.

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/industries` | List available NAICS industry codes |
| `POST` | `/api/predict` | H-1B risk and expected approval-rate prediction |
| `GET` | `/api/companies?industry=54&state=CA` | Company recommendations with progressive relaxation |
| `GET` | `/api/timeline?graduation_date=2025-05-15` | Personalized visa timeline milestones |
